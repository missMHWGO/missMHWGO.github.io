uri与url

http uri url
URI和URL只差一个字母，很多时候都发现用URI的地方貌似也可以用URL= =，所以以前一直都很困惑他两有啥区别，现在有空就记录下来啦~

URI

  • 定义
    URI，Uniform Resource Identifier，翻译成中文是统一资源标识符，用于标识某一互联网资源名称的字符串，该种标识允许用户对网络中（一般指万维网）的资源通过特定的协议进行交互操作。
    讲人话呢，其实就是每个Web服务器资源都有一个名字，这样客户端就可以准确找到它们了，而uri就是这些名字。
  • 类型
    URI包括URL和URN。

URL

  • 定义
    URL，Uniform Resource Locator，翻译成中文是统一资源定位符，是URI最常见的形式。URL 描述了一台特定服务器上某资源的特定位置，它们可以明确说明如何从一个精确、固定的位置获取资源。
  • 语法

其中：
scheme：方案，一般指协议，有http、https、ftp、mailto、telnet等等。
user、password：用户名、密码，一些服务器需要用户输入用户名和密码才允许访问。如ftp协议。
host、port：主机、端口，主机标识了因特网上能够访问资源的宿主机器，可用主机名（域名）或IP地址表示；端口标识了服务器正在监听的网络端口。
path：路径，说明了资源位于服务器的什么地方，通常像一个分级的文件系统路径。
params：参数，向应用程序提供它们所需的输入参数，以便正确地与服务器进行交互，形式为key-value键值对，由“;”将其与URL其余部分分隔开来。例子：ftp://prep.ai.mit.edu/pub/gnu;graphics=true
query：查询字符串，可以通过查询字符串来缩小所请求资源的范围，形式同样为key-value对，之间用字符“&”分隔，由“?”将其与URL其余部分分隔开来。例子：http://localhost/test?id=123&show=false
frag：片段，表示一个资源内部的片段，通常用于在页面中设置“书签”并实现页内跳转。片段出现在URL的最右边，前面有一个字符“#”。注意客户端不会将片段发送到服务器，浏览器从服务器获得整个资源后，会根据片段在页内跳转到指定的位置。

URN

  • 定义
    统一资源名，作为特定内容的唯一名称使用，与目前的资源所在地无关。使用这些与位置无关的URN，就可以将资源四处搬移。通过URN，还可以用同一个名字通过多种网络访问协议来访问资源。为了更有效地工作， URN 需要一个支撑架构来解析资源的位置，而此类架构的缺乏也延缓了其被采用的进度。所以URN仍然处于试验阶段，还未大范围使用。
  • 示例
    可以urn:ietf:rfc:2141来命名因特网标准文档RFC2141，无论它位于何处。
