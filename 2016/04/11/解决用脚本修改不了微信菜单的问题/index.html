解决用脚本修改不了微信菜单的问题

微信 菜单 php
今天参考微信官方文档用php写了一个微信菜单，发现自己根本修改不了菜单的东西。用接口调试工具报错：invalid sub button key size。官网上写的是菜单最多3个，子菜单5个。我很认真地数了一下我的菜单个数——2个，自菜单个数——5个，这超了。。。吗？我读书少你不要欺负我啊= =

解决方法

而我把type的类型写成了click。。。

总结

  1. click是和key匹配的，在微信里的效果就是点击一下菜单，跳出一条消息，显示点击菜单：（你的 key 里的内容）；view和url搭配，效果是点击菜单直接跳转到url那个页面。
  2. 测试过程还发现如果你的代码有bug，那么代码是post不过去的，所以微信的效果还是之前的效果，不会改变。
  3. 代码改变了之后，有的时候要取消关注再关注才会看到效果，特别是改变了菜单内容的时候，有的时候则不需要这样做就能看到效果。
  4. 测试号一天修改菜单的接口调用次数上限是100次，100次，真的只有100次= =所以要节约使用啊~
