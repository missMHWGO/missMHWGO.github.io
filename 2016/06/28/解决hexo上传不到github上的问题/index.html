解决hexo上传不到github上的问题

hexo disconnected github
大概两个星期前我又遇到了刚开始使用hexo的问题，课余时间慢慢捣鼓终于解决了这个问题。

disconnected:No supported authentication methods available (server sent:publickey)

一开始我还暗自庆幸，哈哈，这个问题我当时解决了并且记录了。然而当我用老方法的时候。。。不管用了！！！

看起来应该是key的问题，于是：

1.跑去打开C:\Program Files\TortoiseGit\bin\puttygen.exe生成一个key.
2.跑去打开C:\Program Files\TortoiseGit\bin\pageant.exe，添加之前生成的private key.

然后就是见证奇迹的时刻！！！奇迹就是——disconnected不见了，但是新出来一个问题——

哭晕。。。
这个问题感觉是网络的问题，但是我的电脑是连着网的，不仅连着网还翻着墙。。。
为保证刚刚的操作没有把自己给坑了，于是决定重新生成一个key。

1.进入~/.ssh目录，发现里面居然只有known_hosts。。。我的id_rsa呢呢呢呢呢呢！！！你还我的id_rsa！！！
2.输入ssh-keygen -t rsa -C "wangjiajing@missMHWGO.com"(引号里是github的注册邮箱)重新生成key，其中会提醒输入存储这个key的路径以及passphrase，全部默认回车就好，酱紫key就存在了~/.ssh/id_rsa里并且passphrase为空。
3.输入ssh-agent -s
4.输入ssh-add ~/.ssh/id_rsa,然后就出错了= =
could not open a connection to your authentication agent.
5.输入 eval `ssh-agent -s`
6.输入ssh-add
7.输入clip < ~/.ssh/id_rsa.pub拷贝
8.将这个key粘贴到github里。

然而。。。还是不行！！！
