响应式设计学习（一）

响应式设计 屏幕适配
之前我写过一篇通过识别PC端和移动端跳转到不同的页面来解决显示问题，可是这样就要同时维护好多个版本，既不能偷懒又不能装逼。那么问题来了，到底如何在不同终端设备上显示效果理想的网页？响应式设计就应运而生了~
其实讲人话嘛，就是让不同的网页能够根据设备大大小小的屏幕自动调整显示的样子，而没有做屏幕适配的网页需要通过自由缩放来看到整个页面的内容。阮一峰举了个《福尔摩斯历险记》六个主人公的头像的例子，十分生动形象，我就不赘述了。

移动端的viewport(视区)

为了避免破坏没有针对手机浏览器优化的网页的布局，手机浏览器把页面放在一个虚拟的“窗口”中，通常这个虚拟的“窗口”比屏幕宽，用户可以通过平移和缩放来看网页的不同部分。这个“窗口”就叫viewport。

还有其他的一些属性可供选择：

属性 描述
height 和 width 相对应，指定高度
initial-scale 初始缩放比例，也即是当页面第一次 load 的时候缩放比例
maximum-scale 允许用户缩放到的最大比例
minimum-scale 允许用户缩放到的最小比例
user-scalable 用户是否可以手动缩放
