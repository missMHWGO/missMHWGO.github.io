启用Apache服务器的Gzip压缩

gzip Apache
http压缩，其实就是一种http编码方式，在编码的同时去除了一些冗余，降低了网络传输的字节数，加快了网页加载速度。

http压缩过程

  • Web服务器接收到浏览器的HTTP请求后，检查浏览器是否支持HTTP压缩；
  • 如果支持，Web服务器检查请求文件的后缀名;
  • 如果请求文件是HTML、CSS等静态文件，Web服务器到压缩缓冲目录中检查是否已经存在请求文件的最新压缩文件；
  • 如果请求文件的压缩文件不存在，Web服务器向浏览器返回未压缩的请求文件，并在压缩缓冲目录中存放请求文件的压缩文件；
  • 如果请求文件的最新压缩文件已经存在，则直接返回请求文件的压缩文件。
  • 如果请求文件是动态文件，Web服务器动态压缩内容并返回浏览器，压缩内容不存放到压缩缓存目录中。

首先抓个包，判断浏览器是否支持gzip压缩。

可以看出，浏览器支持gzip和deflate压缩。

gzip和deflate压缩的区别：
两者都是使用Gzip压缩算法，运作原理是类似的，但还是有些差别。

  1. 安装它们的Apache Web服务器版本。Apache 1.x系列没有内建网页压缩技术，所以才去用额外的第三方mod_gzip 模块来执行压缩。而Apache 2.x官方在开发的时候，就把网页压缩考虑进去，内建了mod_deflate 这个模块，用以取代mod_gzip。
  2. 第二个区别是压缩质量。mod_deflate 压缩速度略快而mod_gzip 的压缩比略高。一般默认情况下，mod_gzip 会比mod_deflate 多出4%~6％的压缩量。
  3. 对服务器资源的占用。 一般来说mod_gzip 对服务器CPU的占用要高一些。mod_deflate 是专门为确保服务器的性能而使用的一个压缩模块，mod_deflate 需要较少的资源来压缩文件。这意味着在高流量的服务器，使用mod_deflate 可能会比mod_gzip 加载速度更快。

好，下面就是如何开启啦~

  1. 首先，进入apache目录\bin\apache\apache2.4.9\modules,看是否有mod_deflate.so文件。若没有，则用 /usr/local/apache2/bin/apxs -i -c -a mod_deflate.c 命令安装并编译生成mod_deflate.so文件。
  2. 打开httpd.conf文件，我的路径为\bin\apache\apache2.4.9\conf\httpd.conf，找到LoadModule deflate_module modules/mod_deflate.so这一行，如果前面有#号注释，干掉它！（如果你在第1步的时候安装使用参数-a是没有注释的，如果你用-A就是以注释的形式自动写入httpd.conf文件中）
  3. 现在我们可以对文件压缩规则进行配置啦~
